Edge Rails - Nova opção para o método time_zone_select

22 de Dezembro de 2007

Uma nova opção foi acrescentada ao método time_zone_select, agora você pode indicar um valor padrão para os casos em que o seu usuário ainda não tenha selecionado nada, como a coluna no banco de dados sendo nula, por exemplo. Para isto foi criada a opção :default, então você poderá usar o método das seguintes maneiras:

time_zone_select( "user", "time_zone", nil, :include_blank => true)

time_zone_select( "user", "time_zone", nil, :default => "Pacific Time (US & Canada)" )

time_zone_select( "user", 'time_zone', TimeZone.us_zones, :default => "Pacific Time (US & Canada)")

Onde estamos informando a opção :default deve aparecer com o time zone escolhido já selecionado.

Edge Rails: Rails 2.0.3 deve sair logo

21 de Dezembro de 2007

Mais uma correção importante, o que indica que em breve teremos uma versão 2.0.3 (estou especulando). Há um bug quando se usa rename_column e remove_column em um banco de dados sqlite se a coluna possuir um índice. O erro já foi corrigido no trunk, mas como é algo sério, deve sair em breve mais uma versão do Rails para esta e outras correções que estão sendo feitas.

Ainda aproveitando a deixa do screencast de ontem, o core team do Rails também está usando esta nova funcionalidade no próprio fonte do Rails. Veja a linha 112 do arquivo form_options_helper_test.rb:

# FIXME: fails on Ruby 1.9, probably in html-scanner

E o foco do desenvolvimento ainda é ajustar o Rails para a versão 1.9 do Ruby, muita coisa está sendo feita referente a isto.

RubyGems 1.0.0

20 de Dezembro de 2007

Eric Hodel acabou de anunciar o lançamento do RubyGems 1.0.0 no seu blog. Esta versão inclui em sua maior parte correções de erros e algumas alterações para deixá-lo mais estável.

Para atualizar é simples, se você já estiver usando a versão 0.9.5 basta digitar no terminal:

sudo gem update --system

Claro que se estiver no windows você deve retirar o "sudo".

Se você estiver usando uma versão mais antiga do RubyGems será necessário fazer assim:

sudo gem install rubygems-update

update_rubygems

Edge Rails - Mais Ruby 1.9

19 de Dezembro de 2007

O foco das alterações do Rails ainda é o Ruby 1.9, mesmo os menores detalhes estão sendo analisados para deixar o Rails o mais compatível possível. Detalhes como alterar de ”File.exists?” para ”File.exist?” não estão sendo deixados de fora.

Também, no Ruby 1.9, o módulo Base64 (base64.rb) foi removido, por isto todas as referencias a ele foram substituídas por ActiveSupport::Base64.
